# Готовность к публичному запуску (Leadiya)

Краткий аудит репозитория: что уже есть, что **блокирует** прод, и что желательно до/после релиза.

---

## Критические блокеры (без этого «в прод» небезопасно или неработоспособно)

### 1. Авторизация дашборда

- API защищён: все маршруты под `/api/*` проходят через `authMiddleware` (`apps/api/src/middleware/auth.ts`).
- При **`AUTH_BYPASS=false`** (как должно быть в проде) нужны **Supabase** (`SUPABASE_URL`, ключи и желательно `SUPABASE_JWT_SECRET`) **или** иной источник JWT.
- **Дашборд сейчас не подставляет заголовок `Authorization: Bearer …`** ни в один `fetch`. В коде UI нет интеграции с Supabase Auth (или другим IdP).

**Итог:** в проде либо остаётся небезопасный `AUTH_BYPASS=true` (один общий «пользователь»), либо все запросы получат **401**. Нужны: экран входа, хранение сессии, обёртка над `fetch` / React Query с Bearer-токеном.

### 2. Воркеры и очереди

- `docker-compose.yml` поднимает только **api** и **dashboard**. Сервиса **workers** нет.
- Скрапинг 2GIS, обогащение, Baileys и т.д. завязаны на **BullMQ + Redis** и процесс `apps/workers`.

**Итог:** API может принимать задачи, но **фоновая обработка не выполнится**, если воркеры не запущены отдельно (второй контейнер, systemd, Kubernetes Job, и т.д.). В compose нужен как минимум профиль `workers` или отдельный деплой-манифест.

### 3. Миграции БД

- Перед первым запуском API в новом окружении нужно применить схему Drizzle (`npm run db:migrate` или эквивалент в CI/CD).

**Итог:** зафиксировать в пайплайне шаг «migrate before start» (и откат/бэкап по вашей политике).

### 4. CORS

- В `apps/api/src/server.ts` включён `cors()` **без ограничения origin** — для публичного API это лишняя поверхность атаки (браузерные запросы с любого сайта).

**Итог:** в проде задать allowlist origin’ов (дашборд, при необходимости — отдельный origin API).

## Сильные стороны (уже готово к «коробочному» деплою)

- **Docker:** многостадийная сборка, отдельные образы `api`, `workers`, `dashboard` (`Dockerfile`).
- **Dashboard за nginx:** `apps/dashboard/nginx/default.conf` проксирует `/api/` и `/health` на сервис API в Compose — нет типичной 502 из-за статики без бэкенда.
- **Health:** `GET /health` с `service: leadiya-api` — удобно для балансировщиков и проверок.
- **CI:** сборка, линт, тесты (`.github/workflows/ci.yml`).

## Бизнес-логика и продукт (что уточнить до публичного предложения)

| Тема | Вопрос |
|------|--------|
| **Мультитенантность** | Есть `tenants` и `quotaMiddleware`, но неочевидно, что все маршруты и данные жёстко изолированы по `tenant_id`. Нужен аудит запросов в `packages/db` / роутерах. |
| **Квоты и биллинг** | Stripe-роуты есть; нужна связка «подписка → tenant.active → лимиты экспорта/скрапа». |
| **2GIS / право** | Публичный продукт с автоматическим скрапингом может нарушать ToS 2GIS. Нужна юридическая оценка и формулировки для пользователей. |
| **Данные персональные** | Телефоны, WhatsApp, логи — политика хранения, согласия, удаление по запросу (GDPR/локальные нормы). |
| **Надёжность скрапера** | Капча, блокировки IP, SmartProxy — операционные риски, не только код. |

## Рекомендуемый чеклист перед «публично»

1. [ ] Выключить `AUTH_BYPASS`, настроить Supabase (или другой IdP), **добавить логин в дашборд** и Bearer на все API-вызовы.
2. [ ] Запуск **workers** в проде + мониторинг очереди Redis.
3. [ ] Миграции в пайплайне; бэкапы Postgres.
4. [ ] Ограничить **CORS**; по возможности **rate limiting** на `/api` (особенно login, скрапер, экспорт).
5. [ ] TLS терминация (перед nginx или у хостинга): HTTPS для UI и API.
6. [ ] Секреты только из env / secret manager, не в образе.
7. [ ] (Опционально) Sentry / логирование структурированное.
8. [ ] Расширение Chrome: отдельный план публикации в Web Store, `VITE_PUBLIC_API_ORIGIN` / фиксированный URL API для пользователей.

## Быстрый старт Compose (как в README)

```bash
cp .env.example .env
# Заполнить DATABASE_*, REDIS_URL; для первого теста без JWT — только осознанно: AUTH_BYPASS=true
docker compose up --build
# UI http://localhost:8080 · API http://localhost:3001/health
```

Помните: без воркеров и без миграций «полный» пайплайн лидов не воспроизвести.
